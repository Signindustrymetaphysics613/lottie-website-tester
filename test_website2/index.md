# test_website2 — Page Bloat

A fictional print-gallery site where some pages are tiny and others are
deliberately heavy. Tests Lottie's per-page byte tracking and its
"Heaviest 10 pages" report.

## How to run

```sh
cd test_website2
../copy_lottie_to_test_folders.sh   # or run from project root
python3 -m http.server 8002
# then open http://localhost:8002/lottie.html
```

## Site layout (15 pages, depth 2)

- `index.html`, `about.html`, `gallery.html` — light pages (depth 0/1)
- `prints/print-1.html` … `prints/print-4.html` — small pages (depth 2)
- `prints/print-5.html` … `prints/print-8.html` — medium pages with inline SVG (depth 2)
- `prints/print-9.html` … `prints/print-11.html` — heavy pages with large inline SVG (depth 2)
- `prints/print-12.html` — extra-bloated page that *also* references `large.svg` (depth 2)

Plus three standalone subresources used by the prints:

- `small.svg` (~5 KB)
- `medium.svg` (~190 KB)
- `large.svg` (~2.2 MB)

Generated by `generate_bloat.py` — re-run it from this folder to regenerate.

## What Lottie should find

### All 15 pages crawled successfully (HTTP 200)
No broken links here. All linked SVGs also resolve.

### Heaviest 10 pages (approximate ranking)
Lottie's "Heaviest 10" list should be roughly, from largest down. Sizes are
the HTML alone — referenced SVGs are reported as separate subresources.

1. `prints/print-12.html` — ~600 KB inline + references `large.svg` (~2.2 MB)
2. `prints/print-11.html` — ~340 KB inline
3. `prints/print-10.html` — ~340 KB inline
4. `prints/print-9.html`  — ~340 KB inline
5. `prints/print-8.html`  — ~22 KB inline + references `medium.svg`
6. `prints/print-7.html`  — ~22 KB inline + references `medium.svg`
7. `prints/print-6.html`  — ~22 KB inline + references `medium.svg`
8. `prints/print-5.html`  — ~22 KB inline + references `medium.svg`
9. `prints/print-4.html`  — ~400 B + references `small.svg`
10. `prints/print-3.html`  — ~400 B + references `small.svg`

Exact ordering at the small-page end may shift depending on whether Lottie
counts referenced subresource bytes in the "page" total or separately —
either ordering is acceptable as long as the bloated/heavy pages rank above
the light pages.

### Subresources counted
- `small.svg` referenced by 4 print pages
- `medium.svg` referenced by 4 print pages
- `large.svg` referenced by 1 print page (the bloated one)

### Notes
- No broken links, no broken subresources, no mixed content here.
- Static server cannot simulate slowness, so "slowest pages" is not the
  focus of this site (use a real network to exercise that). Lottie will
  still report some difference because heavier pages take longer to
  download even locally.
